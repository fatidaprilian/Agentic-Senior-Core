# Runtime Token Saver Architecture

Status: planned, unshipped.
Last updated: 2026-05-18.

## Architecture Summary

Build a conservative command wrapper named `ascx`.

`ascx` executes a whitelisted developer command, captures stdout and stderr, writes raw output to a local tee file when needed, applies a safe filter, and returns a compact evidence summary to the agent.

```text
agent command
  -> ascx <tool> <args>
  -> execute original tool
  -> capture stdout/stderr/exit code
  -> tee raw output on failure or truncation
  -> apply safe filter if command is supported
  -> print compact evidence summary
```

## Why `ascx`

`ascx` keeps the runtime concern separate from the governance CLI.

Use:

```bash
ascx git status
ascx git diff
ascx rg "pattern"
ascx npm test
```

`asc` remains the product CLI:

```bash
asc optimize install
asc optimize status
asc optimize off
asc optimize doctor
```

## Runtime Components

### 1. Command Lexer

Parse enough command structure to decide whether compression is safe.

The lexer should detect:

- executable name
- arguments
- environment variable prefixes
- pipes
- redirects
- command substitution
- shell control flow
- unknown shell syntax

If the lexer is unsure, pass through raw behavior.

### 2. Safety Classifier

Classify each invocation:

```text
compressible
passthrough
unsupported
unsafe-for-compression
```

Unsafe examples:

- command output is piped to another process
- command output is redirected to a file
- command uses command substitution
- command is part of a loop
- command output is likely data for another program
- command is explicitly excluded

### 3. Command Adapters

Start with a small whitelist:

- `git status`
- `git diff`
- `git log`
- `rg`
- `ls`
- `tree` when available
- `npm test`
- `npm run build`
- `npm run lint`
- TypeScript compiler output
- common test runner output when detected

Adapters must preserve semantics. If an adapter cannot parse confidently, it should return a safe summary plus a raw tee path, or pass through.

### 4. Tee Writer

Write raw output to a local file when:

- exit code is non-zero
- output is truncated
- compression ratio exceeds a high-risk threshold
- user enables `always` mode

The tee hint must use an absolute path. Avoid `~` shorthand. Avoid ambiguous quoting.

Planned state location:

```text
.agent-context/state/token-saver/
```

For global installs, consider a user data directory later, but project-local state is easier for agent follow-up reads during the MVP.

### 5. Summary Formatter

Every compressed output should include a compact structured footer:

```text
[ascx]
command: npm test
exit: 1
filter: npm-test-summary
raw_tokens: 18420
output_tokens: 2310
reduction: 87%
raw_output: E:\Project\example\.agent-context\state\token-saver\tee\2026-05-18T120000Z-npm-test.log
```

The exact token counting method can start as an approximation and become model-aware later.

## Evidence Summary Contract

Any compressed result must preserve:

- original command
- exit code
- root error
- file path
- line number
- failing test name
- assertion message
- changed file list when relevant
- truncation marker when output is incomplete
- raw output path when detail was removed

It may remove:

- repeated stack frames
- dependency installation noise
- passed test lists
- progress bars
- duplicate warnings
- package manager banners
- repeated identical log lines

It must not remove:

- the only error message
- the only file path
- the only line number
- evidence that multiple failures share a pattern
- security warnings
- migration or destructive-operation warnings

## Hook Strategy

Use progressive support tiers.

Tier 1: explicit wrapper.

```bash
ascx git status
```

Tier 2: instruction fallback generated by `asc optimize install`.

The installer updates project guidance so agents prefer `ascx` for supported shell commands.

Tier 3: agent hooks where safe and documented.

Do not make hook support the MVP. Hook APIs differ, and fragile hooks can create false confidence.

## Supported Agent Strategy

Start with instruction fallback for:

- Codex
- Claude Code
- Cursor
- Windsurf
- Cline/Roo

Add transparent hooks only when each agent's hook contract is clear and tested.

Do not claim transparent support for built-in file tools that bypass shell execution.

## Runtime Language

MVP can start in Node.js because the current repo is a Node.js ESM CLI and publish flow is already npm-based.

A Rust or Go binary can be considered after the MVP if:

- Node startup latency is too high.
- Streaming capture is unreliable.
- Cross-platform packaging becomes painful.
- Parser performance becomes a bottleneck.

Do not pick Rust or Go only because RTK uses Rust. Choose it when measured constraints require it.

## Privacy Boundary

Default:

- no telemetry
- no network upload
- no provider credentials
- no API request proxying
- no command history sync

Local raw tee logs may contain secrets. The doctor command must warn about this and offer cleanup.

## Cleanup

Planned commands:

```bash
asc optimize off
asc optimize doctor
asc optimize clean
```

`off` disables runtime use.

`clean` can be introduced later to remove tee logs, reports, and runtime state.
