# Archived Migration Tooling

These files were operational during the v3-to-v4 transition and the v3-purge readiness work. They are kept here as historical reference only. They are not on any active code path:

- They are not referenced by `package.json` scripts.
- They are not referenced by `npm run validate` or `npm test`.
- Their internal imports are not guaranteed to resolve in their current location; restore them to their original `scripts/` paths if you need to re-run them.

Files:

- `v3-purge-audit.mjs` — Phase 3 purge readiness audit. Used once during v4 cleanup. Output state file `.agent-context/state/v3-purge-audit.json` is no longer generated by default.
- `migrate-rule-format.mjs` and `migrate-rule-format/` — Phase 1 helper that converted legacy v3 prose rule files to the v4 numbered format with YAML frontmatter and stable section IDs. All 15 rule files were migrated successfully and the helper is no longer needed for current work.
- `migrate-rule-format.test.mjs` — Test suite for the migration helper.

If a future format change requires similar migration work, treat these as a starting point only and rebuild against the then-current rule files; do not run them blindly.
