# V1.7 Execution Playbook (H2 2026)

This playbook translates roadmap goals into executable delivery flow.

## Scope

- Frontend system foundation and UX quality gates.
- Governance-grade quality controls for responsiveness, accessibility, and conversion clarity.

## Delivery Phases

### Phase 1: Foundation Baseline
- Establish visual token governance.
- Establish responsive breakpoint and density standards.
- Establish accessibility baseline and reduced-motion policy.

### Phase 2: UX and Conversion
- Improve homepage and docs shell narrative hierarchy.
- Improve onboarding CTA clarity across first-time and existing-project paths.
- Add visual regression guardrails for protected pages.

### Phase 3: Release Gate
- Run frontend usability audit script and archive report artifact.
- Validate checklist completion and release evidence links.

## Required Evidence for Release

- Frontend usability checklist completion.
- Audit report artifact from CI run.
- KPI snapshot for responsiveness and conversion clarity indicators.

## Commands

```bash
npm run validate
npm test
npm run audit:frontend-usability
```
