# Terminology Mapping Reference

This reference defines canonical enterprise terminology and developer-facing aliases used across Agentic-Senior-Core.

## Dual-Term Mapping

| Canonical Enterprise Term | Developer-Facing Alias | Usage Rule |
|---------------------------|------------------------|------------|
| Federated Governance | Federated Rules Operations | Use canonical term in compliance and audit artifacts. |
| Governance Engine | Rules Engine | Use alias in onboarding and day-to-day developer docs. |
| Guardrails | Quality Checks | Use alias in implementation guidance and quickstart docs. |

## First-Mention Rule

In developer-facing docs and runtime onboarding output, first mention must include canonical term in parentheses.

Examples:
- `rules operations context (Federated Governance baseline)`
- `Rules Engine (Governance Engine)`
- `quality checks (guardrails)`

## Compliance Boundary

Compliance and audit artifacts must keep canonical enterprise terminology for legal and operational traceability.
