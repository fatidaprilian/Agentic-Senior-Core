# Agent Portability

ASC supports 16+ AI coding hosts at two tiers.

## Plugin-tier hosts

Full plugin support: hooks inject rules on every session, skills and commands are available on demand. No per-project files needed.

| Host | Plugin path | Hooks | Skills | Commands |
|------|------------|-------|--------|----------|
| Claude Code | `.claude-plugin/` | `hooks/hooks.json` | `skills/` | `commands/*.md` |
| Codex CLI | `.codex-plugin/` | `hooks/hooks.json` | `skills/` | `commands/*.md` |
| Copilot CLI | `.github/plugin/` | `hooks/copilot-hooks.json` | `skills/` | `commands/*.md` |
| Gemini CLI | `gemini-extension.json` | N/A (reads context file) | `skills/` | `commands/*.toml` |
| Devin | `.devin-plugin/` | `hooks/hooks.json` | `skills/` | N/A |
| Hermes | `plugin.yaml` | `__init__.py` (Python) | `skills/` | N/A |
| OpenCode | `.opencode/plugins/` | JS module transform | N/A | N/A |
| OpenClaw | `.openclaw/skills/` | N/A | `.openclaw/skills/` | N/A |
| Antigravity | `.agents/plugins/agentic-senior-core/` | N/A | `.agents/plugins/.../skills/` | N/A |

## Instruction-tier hosts

Single file containing the full rule set. Must be copied into each project. Generated by `asc adapter` or copied manually.

| Host | Adapter path | Format |
|------|-------------|--------|
| Cursor | `.cursor/rules/agentic-senior-core.mdc` | MDC with `alwaysApply: true` frontmatter |
| Windsurf | `.windsurf/rules/agentic-senior-core.md` | Markdown |
| Cline | `.clinerules/agentic-senior-core.md` | Markdown |
| GitHub Copilot (VS Code) | `.github/copilot-instructions.md` | Markdown |
| Kiro | `.kiro/steering/agentic-senior-core.md` | Markdown |

## Hook event mapping

Different hosts use different event names and casing:

| Event | Claude Code | Codex CLI | Copilot CLI |
|-------|------------|-----------|-------------|
| Session start | `SessionStart` | `SessionStart` | `sessionStart` |
| Subagent start | `SubagentStart` | `SubagentStart` | `subagentStart` |

## Antigravity rule format

Antigravity rules are Markdown files with YAML frontmatter. The `trigger` field controls when the rule activates:

| Trigger | Behavior |
|---------|----------|
| `always_on` | Loaded on every session (used by ASC) |
| `manual` | Only when explicitly invoked |
| `model_decision` | Agent decides based on `description` |
| `glob` | Only for files matching `globs` pattern |

ASC uses `trigger: always_on` for the universal rules file. Rules are limited to 12,000 characters per file.

Antigravity plugin bundles live at `.agents/plugins/<name>/` (workspace) or `~/.gemini/config/plugins/<name>/` (global). Each bundle contains `plugin.json`, optional `rules/`, `skills/`, and `hooks.json`.

## Adding a new host

1. Identify the host's plugin manifest format (JSON, YAML, TOML, or Python)
2. Create the manifest file at the host's standard path
3. If the host supports hooks, wire `session-start.js` and `subagent-start.js`
4. If the host supports skills, point to `skills/` or copy SKILL.md files
5. Add the path to `package.json` `files` array
6. Add a test assertion in `tests/adapter.test.mjs`
7. Update this document and `README.md`
