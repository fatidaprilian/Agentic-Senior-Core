# API Contract

## Scope

This project exposes command-line APIs through the `agentic-senior-core`/`asc` main CLI bins, the `ascx` wrapper bin, and repository validation scripts. It does not expose an HTTP API.

## CLI Commands

| Command | Contract |
| --- | --- |
| `agentic-senior-core launch` | Starts the interactive numbered launcher. Running the main CLI without a command enters the same launcher. |
| `agentic-senior-core init [target-directory]` | Initializes the compact governance surface, state files, Compact Natural response guidance, default token optimization state, optional docs bootstrap prompts, and optional MCP helper files. |
| `agentic-senior-core upgrade [target-directory]` | Refreshes managed governance assets, preserves user-owned instruction files, updates onboarding state, backfills missing default token optimization state unless the prior onboarding report opted out, and can prune stale managed `.agent-context/` files. |
| `agentic-senior-core context "<request>" [--json] [--request-id <id>] [--file <path>] [--files <paths>]` | Resolves a request into the Adaptive Context manifest with selected labels, rules, prompts, docs, state, file signals, uncertainty, budget status, and fallback status. |
| `agentic-senior-core optimize [install\|off\|status\|doctor] [target-directory]` | Enables, disables, reports, or diagnoses token optimization state for an initialized repository. |
| `asc optimize status [target-directory]` | Short alias for checking ASCX runtime token saver readiness. |
| `asc optimize doctor [target-directory]` | Short alias for diagnosing ASCX availability, tee write safety, and double-compression warnings. |
| `agentic-senior-core rollback [target-directory]` | Restores files from the latest `.agentic-backup/manifest.json`. |
| `agentic-senior-core mcp` | Starts the local MCP stdio runtime. |
| `agentic-senior-core audit:design-anti-repeat [target-directory] [--json] [--palette-advisory] [--threshold <number>]` | Scans target-project CSS and design-token files against the typography and palette anti-repeat ledger in `docs/design-intent.json`. |
| `agentic-senior-core --version` | Prints the package version. |
| `ascx git status` | Executes `git status`, compresses supported status output, preserves exit code and changed-file evidence, and tees raw output when truncation or high-risk compression occurs. |
| `ascx git diff` | Executes `git diff`, compresses supported unified diff output, preserves changed files, hunk anchors, binary/deleted/generated markers, truncation markers, and tees raw output when detail is removed. |
| `ascx npm test` | Executes `npm test`, compresses TAP-like test output, preserves exit code, failing test names, assertion/root errors, file paths, and writes raw tee output for failed commands. |

`ascx` is an explicit local command wrapper. Unsupported commands and unsafe shell syntax such as pipes or redirects pass through without compression.

## Default Generated Surface

Init and upgrade generate or refresh these default instruction surfaces:

- `AGENTS.md`
- `CLAUDE.md`
- `GEMINI.md`
- `.agent-context/`
- `.agent-context/state/onboarding-report.json`
- `.agent-context/policies/llm-judge-threshold.json`

Default init enables ASCX command guidance through `.agent-context/state/token-optimization.json` and Compact Natural Mode through `.agent-context/prompts/compact-natural-mode.md`. Init and upgrade ensure local `.gitignore` entries for ASCX runtime artifacts such as `.agent-context/state/token-saver/` and `.agent-context/state/token-optimization-report.json`.

`CLAUDE.md` and `GEMINI.md` are import bridges to `AGENTS.md`. Deprecated root and tool-specific instruction files are not generated by default.

`AGENTS.md` instructs compatible agents to run the Adaptive Context resolver before selecting scoped rule files. Agents can pass known touched files through `--file` or `--files` so the resolver can use repository context instead of request text alone. The manifest includes a rule budget so broad requests are visible instead of silently loading every rule family. When the resolver is unavailable, agents fall back to the manual scope map in `AGENTS.md`.

## Public Script Contracts

| Script | Contract |
| --- | --- |
| `npm run sync:adapters` | Rewrites `CLAUDE.md` and `GEMINI.md` to import `AGENTS.md`. |
| `npm run check:adapters` | Fails when the import bridges drift from the canonical shape. |
| `npm run validate` | Runs repository integrity, docs, policy, and coverage checks. |
| `npm test` | Runs the Node test suite. |
| `npm run gate:release` | Runs release-gate and forbidden-content checks. |
| `npm run benchmark:adaptive-context` | Runs the local Adaptive Context fixture benchmark and emits a selected-context manifest report. |
| `npm run benchmark:ascx` | Runs the local ASCX Runtime Token Saver fixture benchmark for `git status`, `npm test`, and unsafe passthrough cases. |
| `npm run benchmark:compact-natural` | Runs the local Compact Natural Mode fixture benchmark for evidence preservation, actionability, register quality, and output reduction. |

## Compatibility

Existing repositories may already contain `.instructions.md`, `.agent-instructions.md`, `.cursorrules`, `.windsurfrules`, `.cursor/`, `.windsurf/`, `.github/`, `.gemini/`, or `.agents/` files from older releases. New installs stop generating those files by default. Upgrade must not overwrite user-owned files that do not contain Agentic-Senior-Core managed markers.
