# V1.8 Operations Playbook

V1.8 focuses on enterprise release governance and supply-chain evidence.

## Objectives
1. Prevent version/changelog drift before release.
2. Produce repeatable machine-readable release evidence.
3. Attach SBOM artifacts to CI runs for audit readiness.

## Release Pipeline
1. Run repository baseline checks.
   - `npm run validate`
   - `npm run test`
2. Run governance checks.
   - `npm run audit:frontend-usability`
   - `npm run gate:release`
3. Generate compliance evidence.
   - `npm run sbom:generate`
4. Review generated artifacts in CI.
   - release-gate report JSON
   - CycloneDX SBOM JSON

## Required CI Workflows
- `.github/workflows/release-gate.yml`
- `.github/workflows/sbom-compliance.yml`

## Operational Controls
- Any failed release gate blocks promotion.
- Any missing required governance artifact blocks promotion.
- SBOM generation failure blocks compliance sign-off.

## Rollback Guidance
1. Revert to previous stable tag if release gate fails after merge.
2. Re-run `npm run validate` and `npm run gate:release` on rollback commit.
3. Regenerate SBOM for rollback commit and attach artifact.

## Ownership
- Engineering owner maintains release checklist and gate script.
- Security owner reviews SBOM artifacts and override expiry hygiene.
- Release manager validates changelog/version consistency before tagging.
